# H-007 — PRD AC-3.2 documenta incorrectamente el invariante de fondos retenidos

## Descripción
El PRD en `docs/enterprise-documentation/02-PRD.md` define:
> AC-3.2: Held funds cannot exceed balance

El comportamiento real implementado en `WalletService.holdFunds()` es diferente (y correcto desde el dominio): cuando se bloquean fondos para una puja, `wallet.balance` disminuye y `wallet.held_funds` aumenta. Después de una puja, `held_funds > balance` es el estado normal esperado.

La restricción real implementada es "no puedes bloquear más de tu balance disponible *en el momento del bloqueo*", lo que la librería `@ironloot/core` verifica via `WalletCalculation.canLockFunds(balance, heldFunds, amount)`.

AC-3.2 tal como está escrito es literalmente falso para cualquier usuario con pujas activas.

## Impacto en el Dominio
Baja severidad operacional (el código es correcto), pero la documentación incorrecta puede:
1. Confundir a desarrolladores que lean el PRD antes de trabajar en el módulo wallet
2. Generar tests incorrectos que intenten verificar AC-3.2 tal como está escrita y fallen falsamente
3. Introducir regresiones si alguien "corrige" el código para alinearlo al PRD en lugar de actualizar el PRD

## Recomendación
Actualizar `docs/enterprise-documentation/02-PRD.md` AC-3.2 para reflejar el invariante real:

**Actual (incorrecto):**

**Correcto:**
