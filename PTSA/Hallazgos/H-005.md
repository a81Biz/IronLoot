# H-005 — CFDI/PAC integration es un stub — IronLoot no emite facturas fiscales (compliance SAT)

## Descripción
El módulo `CfdiModule` / `CfdiService` gestiona el modelo de datos `cfdi_records` en la base de datos pero no realiza ninguna integración real con un Proveedor Autorizado de Certificación (PAC) del SAT mexicano. La emisión de Comprobantes Fiscales Digitales por Internet (CFDI) es un stub declarado.

Este gap está documentado en PRD Out-of-Scope, Foundation Protocol TD-001 (HIGH), HANDOFF.md, y el inventory de servicios — es una limitación conocida y explícita de v1.0.0.

## Evidencia Clave
- **E-007**: `docs/implementation/HANDOFF.md` TD-001 + `docs/enterprise-documentation/02-PRD.md` Out-of-Scope section + `inventory/services.md` CfdiService annotation

## Impacto en el Dominio
**CR-011 violada**: La regla de dominio establece que toda transacción gravable debe producir un `CfdiRecord` válido (folio, UUID, XML). Actualmente las transacciones no generan CFDI válidos aunque el registro en base de datos exista.

En México, el Artículo 29-A del Código Fiscal de la Federación (CFF) y la NOM del SAT exigen CFDI para operaciones de compra-venta. IronLoot como plataforma de subastas que intermedia la venta de bienes entre particulares está sujeta a obligaciones de facturación electrónica al superar umbrales de operación o con compradores que lo requieran.

**Sin CFDI real**: La empresa no puede operar legalmente a escala comercial en México. Los compradores/vendedores no pueden deducir sus operaciones. Hay riesgo de multas del SAT.

## Riesgo Calculado
- Impacto: 2 (alto-dominio — compliance fiscal es bloqueador para operación comercial)
- Probabilidad: 3 (alta — el gap es permanente hasta que se implemente un PAC real)
- Risk_bruto: 6

## Recomendación
Priorizar la integración con un PAC certificado por el SAT (ej: Edicom, SIFEI, Finkok) antes del lanzamiento comercial. El modelo de datos ya existe (`cfdi_records`). El trabajo es implementar `CfdiService.emitCfdi()` con la API del PAC seleccionado, almacenar el XML firmado y el UUID del SAT, y enlazar el `cfdi_record` con los `orders` correspondientes.
