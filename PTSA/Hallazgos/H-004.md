# H-004 — Withdraw: Validación de método de pago comentada (mock) — cualquier referenceId es aceptada

## Descripción
En `WalletController.withdraw()`, el paso de validación del método de pago destino está completamente comentado:

```typescript
// 1. Validate Payment Method (Mock for now, should verify if user has this method registered)
// const method = await this.paymentsService.getUserPaymentMethod(req.user.id, dto.referenceId);
// if (!method) throw new BadRequestException('Invalid payment method');
```

El sistema acepta cualquier `referenceId` arbitrario como destino del retiro sin verificar que ese método de pago le pertenezca al usuario autenticado. El único control activo es el límite diario de 5000 MXN.

## Evidencia Clave
- **E-005**: `src/api/src/modules/wallet/wallet.controller.ts` líneas 43-55 — comentario explicativo del mock

## Impacto en el Dominio
**CR-009 potencialmente violada**: La regla de dominio establece que un retiro debe validar que el método de pago pertenece al usuario. Con este stub, un usuario podría especificar la `referenceId` de otro usuario y redirigir fondos a destinos no autorizados (si el modelo de datos lo permite).

El riesgo real depende de cómo el sistema de pagos asocia `referenceId` con usuarios. Si `getUserPaymentMethod()` es la única barrera (y está comentada), el riesgo es significativo. Si el proveedor de pagos valida la cuenta destino de forma independiente, el riesgo es mitigado por el proveedor externo.

Documentado en Foundation Protocol como TD-003 (MEDIUM). PTSA registra como hallazgo formal porque afecta la integridad financiera.

## Riesgo Calculado
- Impacto: 2 (medio — potencial desvío de fondos a destinatarios no autorizados)
- Probabilidad: 3 (alta — la ruta de código es ejercida en cada retiro, la validación siempre está saltada)
- Risk_bruto: 6

## Recomendación
Implementar `getUserPaymentMethod()` en `PaymentsService` que verifique la asociación usuario↔método de pago en la base de datos. Descomentar la validación en `WalletController.withdraw()`. Añadir tests de integración que verifiquen que un usuario no puede retirar a `referenceId` de otro usuario.

**PT sugerido**: BUG / SECURITY — alta prioridad antes de habilitar retiros en producción. Branch: `fix/PT-XXX-withdraw-payment-method-validation`
