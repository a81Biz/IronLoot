# E-002 — Scheduler lee AUCTION_SOFT_CLOSE_WINDOW_SEC (default 120s) desde SystemConfig

## Contenido capturado
```typescript
async getSoftCloseWindowSec(): Promise<number> {
  return this.systemConfig.getNumber('AUCTION_SOFT_CLOSE_WINDOW_SEC', 120);
}
```

## Observación
El `AuctionSchedulerService` tiene un método `getSoftCloseWindowSec()` que lee la configuración `AUCTION_SOFT_CLOSE_WINDOW_SEC` desde `SystemConfig` con valor por defecto de 120 segundos. Esto es coherente con el PRD y `.env.example`. Sin embargo, el `BidsService` (que aplica la extensión durante una puja) usa `EXTENSION_MS = 5 * 60 * 1000` (300s) hardcodeado y NO invoca este método. Las dos implementaciones son inconsistentes.
