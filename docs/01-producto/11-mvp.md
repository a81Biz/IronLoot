# 11. Alcance del Producto Mínimo Viable (MVP)

Esta sección define con claridad **qué incluye y qué no incluye** el primer lanzamiento funcional de Iron Loot.
El objetivo del MVP es **validar el núcleo del producto** —confianza, subasta funcional y experiencia clara— sin introducir complejidad innecesaria.

El MVP no es una versión incompleta, sino una **versión intencionalmente acotada**, diseñada para aprender, iterar y crecer sobre bases sólidas.

---

## 11.1 Funcionalidades incluidas en el MVP

El MVP de Iron Loot debe incluir las siguientes funcionalidades esenciales:

* Entrada pública con:

  * Home pública.
  * Escaparate de subastas.
  * Página pública de detalle del artículo.
* Registro y autenticación de usuarios.
* Dashboard del usuario (rol cliente).
* Activación del rol de vendedor.
* Dashboard del vendedor.
* Creación y publicación de subastas.
* Visualización de subastas activas y cerradas.
* Sistema de pujas funcional conforme a las reglas básicas.
* Flujo completo de:

  * Cierre de subasta.
  * Pago.
  * Entrega.
  * Calificación.
* Sistema básico de reputación.
* Flujo básico de disputa y soporte.

Estas funcionalidades permiten ejecutar **el ciclo completo de una subasta**, desde la exploración hasta la finalización de la transacción.

---

## 11.2 Funcionalidades excluidas del MVP

Para mantener el foco y reducir riesgos, las siguientes funcionalidades **no forman parte del MVP**:

* Herramientas avanzadas para vendedores profesionales.
* Analíticas detalladas y reportes avanzados.
* Funcionalidades de automatización compleja.
* Servicios adicionales de valor añadido.
* Experiencias avanzadas no críticas para el flujo base.

Estas funcionalidades se consideran parte de fases posteriores del producto.

---

## 11.3 Funcionalidades preparadas para fases posteriores

Aunque no se incluyan en el MVP, el diseño del producto debe **anticipar** la incorporación futura de:

* Funcionalidades avanzadas para vendedores frecuentes.
* Experiencias personalizadas para compradores recurrentes.
* Mejores herramientas de seguimiento y control.
* Ampliaciones del sistema de reputación.
* Evolución del sistema de disputas.

El MVP debe construirse de forma que **no bloquee ni contradiga** estas futuras extensiones.
